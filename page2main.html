9. Describe how you would go about testing digital solutions

The testing of digital solutions is conducted in two ways. The first type is the testing of the unit, or code. A version of unit testing is usually carried out throughout a project, regularly and at various stages. Developers will usually test their code when completing each section of the project and rectify any errors by de-bugging. Once a project has been completed, the code will be run and tested to confirm that it works without any errors and the result is the desired outcome. With time being limited on the course with Code Nation, official unit testing was not permitted, however I was able to test my code in various way to confirm that my programming was at an funcational standard. The second type of testing is user testing. This confirms that the digital solution is functional, easy to navigate and use and that the user experience is as desired. Again, as time was limited on the Code Nation course I was unable to officially user test my website. However, it will be user tested by my tutor and also tested when it is graded.

11. Copyright and other constraints affecting websites.

My website and the content that I share will be created by myself, therefore I would think there would be an issue of copyright infringement. UK Copyright law states that any creation, such logo’s, art, music, works of writing and photography etc, is automatically protected by law. Therefore, anything that I create is legally mine and cannot be infringed upon or used by anybody else. This is something that website owners need to be very careful about especially when sourcing content and images for their site. Permission must be given from the original creator of the material. Open source content is free and can be used by anyone for commercial purposes, which makes it an option to source content and images for my sight. Another regulation which website owners can be restricted by such GDPR (General Data Protection Regulations), which grants more control in favour of the user, regarding the way in which websites collect, store and use data. Having a privacy policy on the site can encourage trust and increase engagement. Users want to feel safe and being as open as possible can enhance trust and loyalty. [1]

12. Access issues for people with disabilities.

People who suffer from disabilities such as impaired motor skills, hearing and vision impairment, dyslexia and who might be epileptic, might have restricted access to my website. For example, if somebody has difficulty hearing or seeing, they may struggle to engage with the pages, since they cannot see or hear the content. The Equality Act 2010 and WCAG and The Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018, state that non-public websites are exempt from compliance. I would like to make my website more accessible and so I would try and implement some functions to allow the user to adjust the written content font size and colour, to add subtitles to any videos that I create and share and exclude flashing images on my website. 13. The appropriate filetypes for websites.

The appropriate file types to use for my website will be; CSS, which stands for cascading Stylesheet and HTML (Hyper Text Mark-up Language). These files will be used to programme and style the website pages. For images, the preferred choice is PNG (Portable Network Graphics). PNG is typically a bigger file size than its common counterpart JPG, short for JPEG (Joint Photographic Experts Group) file, however PNG does not loose quality when saved as it is ‘lossless’, unlike the ‘lossy’ JPG file. AVI (Audio/Video Interleaved) is a file that is suitable for websites as a video format which is supported by the windows platform. They do not stream; therefore, the whole video would need to download before the user can view it. A video file format choice that allows for streaming could be MPEG (Motion Picture Experts Group File) MP3 is a good quality sound file which can be used for any possible pod casts in the future, should I choose to produce any for my website blog. [3]
